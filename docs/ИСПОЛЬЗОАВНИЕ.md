ИНСТРУКЦИЯ ПО ЭКСПЛУАТАЦИИ И ПРОВЕРКЕ
=======================================

Первый запуск
-------------
Требование
    Убедитесь, что СУБД PostgreSQL запущена и доступна локально
    (`localhost:5432`) с пользователем по умолчанию `postgres`.
Действие
    Запустите исполняемый файл приложения (`CourseProject.exe`).
Результат
    При первом запуске программа автоматически создаст базу данных
    `course_db`, необходимые таблицы и учетную запись администратора.
    Также будет создан зашифрованный файл курса `course.bin` в
    системной папке данных приложения.

Роли и учетные данные
-----------------------
В системе предусмотрено две роли:

1.  **Администратор:**
    -   **Назначение:** Управление курсом и мониторинг успеваемости
        студентов.
    -   **Учетные данные (по умолчанию):**
        -   **Логин:** `admin`
        -   **Пароль:** `admin`

2.  **Студент:**
    -   **Назначение:** Прохождение учебного курса.
    -   **Учетные данные:** Создаются самостоятельно через интерфейс
        регистрации.

Сценарий работы администратора
-------------------------------
1.  **Вход в систему:**
    -   Введите логин `admin` и пароль `admin`, нажмите «Войти».
    -   Откроется главное окно с двумя вкладками: «Студенты» и
        «Редактор курса».

2.  **Вкладка «Студенты»:**
    -   **Просмотр:** Отображается таблица всех зарегистрированных
        пользователей.
    -   **Поиск:** Введите логин или его часть в поле «Поиск» для
        фильтрации списка.
    -   **Сортировка:** Нажмите на заголовок столбца для сортировки данных.

3.  **Вкладка «Редактор курса»:**
    -   **Выбор главы:** В списке слева выберите главу для редактирования.
    -   **Редактирование:** В полях справа измените заголовок или
        содержимое главы.
    -   **Сохранение:** Нажмите кнопку «Сохранить изменения». Данные будут
        перезаписаны в зашифрованный файл `course.bin`.

4.  **Создание отчета:**
    -   На вкладке «Студенты» нажмите кнопку «Создать отчет».
    -   Будет сгенерирован текстовый файл (`.txt`), содержащий сводку
        по успеваемости всех студентов (количество пройденных глав,
        дата последней активности).

Сценарий работы студента
-------------------------
1.  **Регистрация:**
    -   В окне авторизации введите желаемый логин и пароль (минимум 3
        и 4 символа соответственно).
    -   Нажмите кнопку «Регистрация».
    -   После сообщения об успехе можно войти в систему с новыми данными.

2.  **Обучение:**
    -   После входа открывается окно с теоретическим материалом первой
        (или первой непройденной) главы.
    -   Изучите материал и нажмите кнопку «Пройти тест».

3.  **Тестирование:**
    -   Отвечайте на вопросы, выбирая один из вариантов и нажимая кнопку
        «Ответить».
    -   **Логика:**
        -   **Успех:** Если все ответы верны, тест считается пройденным.
            Прогресс сохраняется, открывается следующая глава.
        -   **Ошибка:** При неверном ответе выводится предупреждение.
        -   **Провал:** При совершении **3-й ошибки** тест считается
            проваленным. Студент принудительно возвращается к изучению
            теории этой же главы.
